## 仓位参数（position-calculator.ts）

| 参数 | 当前值 | 调参历史 | 理由 | 敏感度 |
|------|--------|---------|------|--------|
| **baseWeight 系数** | `0.022` | 0.008 → 0.015 → **0.022** | fitness×系数=基础仓位。死循环诊断后调高，让 fit7-8 拿 14-17% | 🔴 极高（直接决定仓位大小） |
| **波动率折扣档** | `0.6 / 0.8 / 1.0` | 未调 | >4%/日→0.6，2-4%→0.8，<2%→1.0，未知→0.6 | 🟡 中（影响题材股仓位） |
| 波动率分界点 | `2% / 4%` | 未调 | 大盘/成长/题材的分界 | 🟡 中 |
| single_name 截断 | 在 config 里 | 见下 | 公式值超过上限则截断 | — |

### baseWeight 速算表（当前系数 0.022）

```
fitness × 0.022 = 基础仓位（未乘波动率折扣）
 3 →  6.6%
 5 → 11.0%
 7 → 15.4%
 8 → 17.6%  （×0.8成长股折扣 = 14.1%）
 9 → 19.8%
10 → 22.0%  （顶到 single_name 上限）
```

## 约束参数（rebalance-types.ts DEFAULT_REBALANCE_CONFIG）

| 参数 | 当前值 | 调参历史 | 理由 | 敏感度 |
|------|--------|---------|------|--------|
| **single_name** | `0.22` | 0.10 → 0.15 → **0.22** | 单仓上限。集中定位，让 fit10 不被截断 | 🔴 高（决定能否重仓） |
| **single_sector** | `0.25` | 未调 | 单行业上限（一级聚合）。阶段 13 去共振+top15 后基本不再触发（持仓已自然跨 4 行业）；之前电子偏科时频繁撞线 | 🟡 中（极端主线集中时仍可能触发） |
| **max_positions** | `5` | 无 → **5**（新增） | 持仓数上限。落实"3-5 只"集中定位——之前只是 prompt 软引导，LLM 无视买到 7-8 只触发死亡螺旋 | 🔴 高（治本，限持仓失控） |
| **daily_turnover** | `0.50` | 0.40 → **0.50** + 算法改单向 | 日换手上限。算法从双向 sum(\|delta\|) 改成单向 max(买,卖)，修满仓换仓死亡螺旋 | 🔴 高（双向算法时满仓必卡死） |
| **cash_reserve** | `0.03` | 0.05 → **0.03** | 现金下限。趋势要在场，少留现金 | 🟢 低（影响可花现金池） |
| **initial_stop_drawdown** | `0.07` | 未调 | 建仓回撤止损阈值。-7% 触发 | 🟡 中（止损灵敏度） |
| **initial_stop_days** | `3` | 未调 | 建仓后止损窗口（天）。⚠️ 生益第6天破位未触发，可能偏短 | 🟡 中（待 A/B） |
| anti_churn_days | `7`（自然日≈5 交易日） | 未调 | 持仓<7天禁 SELL（止损/止盈信号除外）。用自然日，跨周末白占 | 🟡 中（防 churn vs 灵活） |
| **take_profit_threshold** | `0.15` | 无 → **0.15**（新增） | 止盈豁免阈值。locked 期内浮盈≥15% 允许突破锁卖出——落袋为安不是 churn。修豫光 +7.6% 想止盈被锁挡死 | 🟡 中（太低→锁形同虚设，太高→锁死该止盈的） |
| max_revise_retries | `3` | 2 → **3** | 违反约束后 LLM 重出次数。+1 给换手/持仓数违规更多收敛机会 | 🟡 中（影响否决频率） |
